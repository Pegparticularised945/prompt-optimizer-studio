# Prompt Quality Rubric (0-100)

用于 `Review` 模式的量化打分。

## Scoring Dimensions

1. 目标清晰度（15）
- 是否明确“要解决什么问题”和“成功标准”。

2. 输入约束完整度（10）
- 是否给出必要变量、边界条件、数据来源限制。

3. 输出契约明确度（15）
- 是否规定输出结构、字段、格式、顺序。

4. 逻辑闭环（15）
- 是否有可追踪流程（分析 -> 决策 -> 交付）。

5. 可执行性（10）
- 是否可直接运行，而非概念性口号。

6. 鲁棒性（10）
- 是否处理缺失变量、异常输入、冲突约束。

7. 防幻觉与证据约束（10）
- 是否要求事实隔离、来源边界或可验证依据。

8. 反死胡同能力（10）
- 是否有熔断、回退、重试或改道机制。

9. 可迭代性（5）
- 是否易于增量优化，不需每次重写。

## Decision Threshold

- `>= 85`: 高质量，只做微调。
- `70-84`: 中质量，保留结构并做定向修复。
- `< 70`: 低质量，切换统一框架重构。

## Dead-End Signals

命中任意 2 条即可判定“高风险死胡同”：

- 只给目标，不给验收标准。
- 变量多但未定义来源。
- 输出要求与输入条件冲突。
- 没有异常分支或回退策略。
- 要求过长链路但缺少阶段性验证点。

## Minimum Viable Experiment (MVE)

优化后必须附带 1 个最小验证实验：

- 目标：在最短路径验证“方向是否正确”。
- 输入：最小样本数据或单场景条件。
- 通过标准：1-2 条可判定指标。
- 失败动作：立即缩小范围或切换策略。
