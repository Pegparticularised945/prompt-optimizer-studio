# Dashboard Delivery-First Design

**Context**

当前首页把全部任务平铺在一个队列里，`failed`、`cancelled`、旧 `completed` 会和当前真正需要关注的任务混在一起，容易让首页重新变脏。用户已经确认希望首页优先呈现“现在该处理什么”和“刚产出了什么”，把历史任务收进去。

**Goal**

把首页改成“交付物优先”的任务看板：主视图优先显示活跃任务、排队中任务和最近完成结果；历史任务收进默认折叠区。

**Confirmed Product Decisions**

- 首页主区分为四段：
  - 活跃任务：`running / paused / manual_review`
  - 排队中：`pending`
  - 最近完成：最近 `3` 条 `completed`
  - 历史任务：其余 `completed / failed / cancelled`
- 历史任务默认收起。
- 不改后端状态机，不改任务详情页，不改 worker 链路。

**Architecture**

展示层只需要一个纯函数，把任务列表按状态和时间切成四组。首页组件继续从 `/api/jobs` 拉取完整列表，但渲染时先做分组：

- 主区只渲染活跃任务、排队中、最近完成
- 历史任务渲染到原生 `<details>` 折叠区

这样实现最小、测试最稳，也不会改变现有数据流和接口。

**Presentation Rules**

- 统计卡改为：
  - 活跃任务
  - 排队中
  - 最近完成
  - 历史任务
- 每个分组都有简短说明和空状态。
- 历史任务卡片降权显示，但仍保留进入详情的链接与关键元数据。

**Testing**

- 新增纯函数测试锁定：
  - 活跃任务分类正确
  - 排队中单独分组
  - 最近完成只取最近 `3` 条
  - 剩余终态任务都进入历史任务

**Non-Goals**

- 不做状态筛选器
- 不新增归档/删除功能
- 不改变任务排序来源，只做前端分组与展示层重排
