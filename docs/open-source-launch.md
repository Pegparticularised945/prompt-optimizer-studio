## 仓库名

`prompt-optimizer-studio`

## 定位文案

### GitHub About

自动化提示词优化流水线，支持人工引导并交付可直接复制的最终完整提示词。

### 简短介绍

Prompt Optimizer Studio 把提示词打磨过程做成了一个可操作的流水线。你先给初版提示词，再让 optimizer 和 reviewer 自动多轮推进；如果方向偏了，人可以立刻介入纠偏，最后得到的是一份真正能拿去用的完整 prompt。

## 重点信息

- 自动化、多轮、流水线式优化提示词
- 人工始终在回路里，而不是只能事后重来
- 最终交付物是最新完整提示词，不是 diff 日志
- 轮次历史、偏题检查和停止规则都可见
- 任务创建、任务详情与运行参数链路更完整
- 支持多 provider / 多模型接入与 Docker 自托管

## Release 历史

### v0.1.7

本次发布形态：

- 当前版本仍是 **Self-Hosted / Server Edition（自托管服务端版）**。
- 数据保存在运行这套应用的机器或部署环境上。
- 本次版本名统一为 **`v0.1.7`**，不使用 `hotfix` 作为对外发布名。
- 本次版本聚焦公开版主路径的体验修复、结构治理与兼容性加固。

本次发布重点：

1. **结果台与轮次展示更容易理解**
   - 轮次卡片与任务详情文案整体收口为更直白的人话。
   - `上轮提示词评分`、`这版后来已经评过分`、`这版要到下一轮才会评分` 等表达更符合当前产品语义。
   - 空态、占位式 MVE 与原始上游错误展示已进一步收口。

2. **连续三轮过线的停止逻辑修正**
   - 修复了满足连续过线后仍继续开新一轮的问题。
   - 修复了满足停止条件但同轮 optimizer 未产出新版本时，整条任务被错误打成 failed 的问题。
   - 现在会优先按“连续 3 轮过线”的当前产品规则正确收口最终交付。

3. **长期规则与任务评分标准链路更一致**
   - 自动运行中的任务详情也保留了“调整长期规则”的入口。
   - 待生效引导可先生成长期规则草稿，再由用户确认保存。
   - 长期规则、待生效引导与任务级评分标准之间的关系说明更清楚。

4. **结构治理与兼容链路正式收口**
   - `jobs / runtime / providers / settings / prompt-pack / db / goal-anchor` 模块边界已明确。
   - OpenAI-compatible `/responses` 回退回归已修复并补齐验证。
   - 新增 `check:architecture` 与模块边界测试，避免后续回退到根级隐式依赖与兼容桥接层。

5. **默认 Skill / Rubric 公开版收口**
   - 默认 prompt-pack 已移除内部实验代号与 lineage 标记。
   - 保留了 `Create / Debug / Review` 的模式保真约束，减少为了追求更短而把任务模式写塌的风险。

### v0.1.6

本次发布形态：

- 当前版本仍是 **Self-Hosted / Server Edition（自托管服务端版）**。
- 数据保存在运行这套应用的机器或部署环境上。
- 未来可能会有独立的 `Web Local Edition`，但它不属于这次发布内容。

本次发布重点：

1. **收紧 provider 重试边界**
   - 不再对 `auth_unavailable` 这类明确的上游认证不可用错误做盲重试。
   - 不再把 generic `INTERNAL_ERROR` 默认当成可重试错误反复放大。
   - 只有 timeout、gateway、upstream、EOF、network 等明确瞬时故障才继续重试。

2. **更真实地暴露运行时失败**
   - 当上游 provider 真的不可用时，任务现在会更快失败，而不是长时间卡在 `round 0`。
   - 这让控制室看到的失败更接近真实原因，也更容易判断该重试 provider 还是检查配置与网关。

3. **补齐回归覆盖**
   - 新增了 `auth_unavailable 500` 不应重试的回归测试。
   - 新增了 generic thrown `INTERNAL_ERROR` 不应重试的回归测试。
   - 同时保留了 `504` 与 timeout/network 类故障继续重试的覆盖，避免把容错能力一起删掉。

### v0.1.5

本次发布形态：

- 当前版本仍是 **Self-Hosted / Server Edition（自托管服务端版）**。
- 数据保存在运行这套应用的机器或部署环境上。
- 未来可能会有独立的 `Web Local Edition`，但它不属于这次发布内容。

本次发布重点：

1. **控制室状态展示更准确**
   - 修复了任务控制室把“尚未产生成绩”的任务显示成 `0.00` 的问题。
   - 现在，无有效成绩的任务会更准确地显示为 `—` 与“未产生成绩”。
   - 最近结果与历史任务中的最佳分数展示也一起对齐。

2. **基础设施错误提示更易理解**
   - 补充识别了 `stream error`、`INTERNAL_ERROR`、`received from peer`、`server_error` 等上游失败信号。
   - 这类问题现在会被统一展示为可重试的基础设施 / provider 层错误。
   - 不再把难懂的原始错误载荷直接甩给用户。

3. **回归覆盖补齐**
   - 补充了无成绩任务卡片展示的回归测试。
   - 补充了 stream/internal provider 错误归类的回归测试。
   - 让这次控制室小维修更适合作为稳定的公开补丁发布。

### v0.1.4

本次发布形态：

- 当前版本仍是 **Self-Hosted / Server Edition（自托管服务端版）**。
- 数据保存在运行这套应用的机器或部署环境上。
- 未来可能会有独立的 `Web Local Edition`，但它不属于这次发布内容。

本次发布重点：

1. **任务创建交互修复**
   - 修复了首页投递台收起后无法再次展开的问题。
   - 修复了投递台收起后页面可能被隐藏层挡住、导致无法继续点击的问题。
   - 首页创建新任务的主路径更稳定。

2. **错误提示更清楚**
   - 对 `504 Gateway Timeout`、`Bad Gateway`、`Cloudflare`、`upstream` 等网关与上游故障做了更一致的识别。
   - 这类问题现在会被显示为可重试的基础设施错误，而不是直接向用户暴露原始错误页。
   - 更容易判断当前应该直接重试，还是进一步检查 provider / gateway / 网络。

3. **多轮运行更稳**
   - 当任务已经产出可用结果时，即使后续轮次遇到基础设施故障，也会尽量保留当前结果与进度。
   - `step` 模式会软着陆到 `paused`。
   - `auto` 模式会软着陆到 `manual review`。
   - 不再因为后续一轮网关异常，就把整条任务简单视为完全失败。

4. **回归覆盖补齐**
   - 补充了基础设施错误归类测试。
   - 补充了多轮运行 fail-soft 软着陆测试。
   - 补充了首页投递台收起 / 再展开行为的回归测试。

### v0.1.3

本次发布形态：

- 当前版本仍是 **Self-Hosted / Server Edition（自托管服务端版）**。
- 数据保存在运行这套应用的机器或部署环境上。
- 未来可能会有独立的 `Web Local Edition`，但它不属于这次发布内容。

本次发布重点：

1. **运行参数更可控**
   - 所有模型现在都可以配置推理强度。
   - 任务运行参数在设置、创建、任务详情、API 与数据库之间的保存链路更完整。
   - 任务级模型与推理强度的调整可以被更清楚地追踪。

2. **任务创建与详情体验改进**
   - 首页投递台创建任务时可直接带入关键运行参数。
   - 任务详情页把低价值的摘要项替换成更有用的 `推理强度` 信息。
   - 模型选择、参数编辑与结果查看的主路径更统一。

3. **结果与状态展示更清楚**
   - 无分数、失败态、最佳分数等展示更合理。
   - 更容易判断任务当前停在什么状态、是否已有可比较结果。
   - 一批提示词理解与目标锚点生成的小修复也一起纳入公开主路径。

4. **稳定性与审计链路补齐**
   - settings / jobs / API / UI / DB 之间的参数快照更完整。
   - 修补了一批围绕任务创建、详情展示、运行参数同步与运行稳定性的通用问题。
   - 保持公开版主路径对齐，但不引入实验台、私有 skill / rubric / prompt-pack 语义变更。

### v0.1.0

- 首个公开版本。
- 自动化提示词优化流水线：系统会按轮次持续推进，而不是只做一次改写。
- 最终完整提示词优先：当前最新完整 prompt 始终可见、可复制。
- 人工控制闭环：支持暂停任务、补充下一轮引导、继续一轮、恢复自动运行。
- 目标锚点防漂移：尽量让多轮优化持续贴合原始任务意图。
- reviewer 隔离：reviewer 只看当前候选稿和评分规则，不看历史聚合问题列表，也看不到人工引导原文。
- Docker 自托管就绪：内置 Dockerfile、Compose 路径、持久化卷约定，以及 `/api/health` 健康检查。
- `AGPL-3.0-only` 协议：如果你修改后继续拿去做在线服务，对应源码也必须向用户提供。
