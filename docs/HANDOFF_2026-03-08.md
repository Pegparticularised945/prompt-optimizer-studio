### 2.2 流程架构已从旧版切换到新版
当前不是“多 judge 并行”，而是：
- `optimizer`：负责改 prompt
- `reviewer`：负责独立复核和打分
- 同一个任务模型别名会同时用于 optimizer 与 reviewer
- reviewer 每次都是全新对话，只看当前 prompt 和评分规则
- optimizer 只看：
  - 当前 prompt
  - 最近一轮 reviewer 提炼出来的精简 patch
- 现在已经实现：
  - 轻上下文
  - feedback 压缩
  - timeout 拉长
  - 单 reviewer 顺序复核
  - `passStreak` / `lastReviewScore` / `lastReviewPatch`

## 6. 当前明确未完成事项

### 6.1 用户已确认但尚未实现
用户已确认下一步要做：
- `暂停`
- `继续一轮`
- `恢复自动运行`
- 任务级 `最大轮数覆盖`

已讨论并确认的产品方向：
- `继续一轮`：只跑一个完整循环，然后自动回到暂停
- `恢复自动运行`：持续跑到完成、失败、人工复核、取消或达到最大轮数
- `暂停`：running 时协作式暂停，当前轮结束后停下；非 running 直接暂停
- `最大轮数`：设置页保留全局默认，任务详情页支持单独覆盖
